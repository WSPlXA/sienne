# oauth2-sienne-idp 接入手册

本文面向需要接入本 IdP（OAuth2 + OIDC）的外部开发者，优先给出可直接跑通的最小闭环。

## 1. 接入前先确认

- 核心端点：
  - `/.well-known/openid-configuration`
  - `/oauth2/authorize`
  - `/oauth2/token`
  - `/oauth2/userinfo`
  - `/oauth2/introspect`
  - `/oauth2/jwks`
  - `/connect/logout`
  - `/oauth2/device/authorize`
  - `/device`
- 浏览器交互接口（`/login`、`/consent`、`/logout`、`/connect/logout`、`/device` POST）要求 CSRF 双提交（Cookie + Header 或 Body）。
- `return_to` 只允许本地相对路径（必须以 `/` 开头，不能是绝对 URL）。

## 2. 外源 App 如何构造回调并交换令牌（最小闭环）

### 2.1 先构造授权请求

外源 App 在跳转授权前要准备：

- `state`：随机串，回调时必须校验一致
- `code_verifier`：随机串
- `code_challenge`：`BASE64URL(SHA256(code_verifier))`
- `redirect_uri`：必须与平台登记值完全一致（协议/端口/路径全匹配）

请求模板：

```text
GET {BASE_URL}/oauth2/authorize
  ?response_type=code
  &client_id={CLIENT_ID}
  &redirect_uri={URL_ENCODED_REDIRECT_URI}
  &scope=openid%20profile%20email%20offline_access
  &state={RANDOM_STATE}
  &nonce={RANDOM_NONCE}
  &code_challenge={CODE_CHALLENGE}
  &code_challenge_method=S256
```

### 2.2 回调接收规范

成功回调：

```text
{REDIRECT_URI}?code={AUTH_CODE}&state={RANDOM_STATE}
```

失败回调：

```text
{REDIRECT_URI}?error=...&error_description=...&state=...
```

回调处理必须做：

- 校验 `state`
- 先判断是否携带 `error`
- 读取一次性 `code` 后立即换 token

### 2.3 交换令牌

机密客户端（`client_secret_basic`）：

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/token" \
  -H "Authorization: Basic $(printf 'web-client:secret123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=authorization_code" \
  --data-urlencode "code=AUTH_CODE" \
  --data-urlencode "redirect_uri=http://localhost:3060/callback" \
  --data-urlencode "code_verifier=YOUR_CODE_VERIFIER"
```

公共客户端（`token_endpoint_auth_method=none`）：

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=authorization_code" \
  --data-urlencode "client_id=mobile-public-client" \
  --data-urlencode "code=AUTH_CODE" \
  --data-urlencode "redirect_uri=myapp://callback" \
  --data-urlencode "code_verifier=YOUR_CODE_VERIFIER"
```

当前实现对 `authorization_code` 的硬约束：

- 必填 `grant_type=authorization_code`
- 必填 `code`
- 必填 `redirect_uri`
- client 要求 PKCE 时必填 `code_verifier`

### 2.4 换 token 成功后的字段

- 必有：`access_token`、`token_type`、`expires_in`
- scope 含 `openid`：返回 `id_token`
- 满足 refresh 条件：返回 `refresh_token`

refresh 返回条件（缺一不可）：

- client 允许 `refresh_token` grant
- client 的 `refresh_token_ttl_seconds > 0`
- scope 包含 `offline_access`

## 3. 本地快速启动（接入调试）

在仓库根目录执行：

```powershell
$env:IDP_HTTP_PORT="8080"
$env:IDP_ISSUER="http://www.idpsienne.uk"
$env:IDP_FORCE_MFA_ENROLLMENT="false"
$env:IDP_TOTP_SECRET_ENCRYPTION_KEY="replace-with-32+chars-secret"
docker compose -f compose.quickstart.yaml up -d
```
## 4 共有信息提取
验证：

```bash
curl -sS http://www.idpsienne.uk/healthz
curl -sS http://www.idpsienne.uk/.well-known/openid-configuration
```

注意：

- 不设置 `IDP_ISSUER` 时，默认是 `http://www.idpsienne.uk`，discovery 返回地址会和本地访问地址不一致。
- 不设置 `IDP_FORCE_MFA_ENROLLMENT=false` 时，首次登录会被强制走 TOTP 绑定。




客户端示例：

- `web-client`（`authorization_code` + `refresh_token`，要求 PKCE，secret=`secret123`）
- `mobile-public-client`（`authorization_code` + `refresh_token`，`token_endpoint_auth_method=none`）
- `service-client`（`client_credentials`，secret=`service123`）
- `legacy-client`（`password` + `refresh_token`，secret=`service123`）
- `tv-client`（`urn:ietf:params:oauth:grant-type:device_code`，secret=`service123`）

## 5. 首选流程：Authorization Code + PKCE

### 5.1 发起授权（浏览器跳转）

```text
GET /oauth2/authorize
  ?response_type=code
  &client_id=web-client
  &redirect_uri=http%3A%2F%2Flocalhost%3A3060%2Fcallback
  &scope=openid%20profile%20email%20offline_access
  &state=YOUR_STATE
  &code_challenge=YOUR_CODE_CHALLENGE
  &code_challenge_method=S256
```

说明：

- 未登录会 302 到 `/login?return_to=...`
- 未同意授权会 302 到 `/consent?return_to=...`
- 成功后回调到 `redirect_uri?code=...&state=...`

### 5.2 Refresh Token Rotation

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/token" \
  -H "Authorization: Basic $(printf 'web-client:secret123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=refresh_token" \
  --data-urlencode "refresh_token=OLD_REFRESH_TOKEN"
```

## 6. 机器对机器：Client Credentials

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/token" \
  -H "Authorization: Basic $(printf 'service-client:service123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=client_credentials" \
  --data-urlencode "scope=internal.api.read"
```

## 7. 兼容流程：Password Grant（仅遗留系统）

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/token" \
  -H "Authorization: Basic $(printf 'legacy-client:service123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=password" \
  --data-urlencode "username=alice" \
  --data-urlencode "password=alice123" \
  --data-urlencode "scope=openid profile offline_access"
```

## 8. 设备流：Device Authorization Grant

### 8.1 设备端申请 device_code

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/device/authorize" \
  -H "Authorization: Basic $(printf 'tv-client:service123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "scope=openid profile"
```

返回字段：`device_code`、`user_code`、`verification_uri`、`verification_uri_complete`、`expires_in`、`interval`。

### 8.2 用户浏览器授权

- 打开 `verification_uri_complete`
- 登录后提交 `approve` 或 `deny`

### 8.3 设备端轮询 token

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/token" \
  -H "Authorization: Basic $(printf 'tv-client:service123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=urn:ietf:params:oauth:grant-type:device_code" \
  --data-urlencode "device_code=DEVICE_CODE"
```

可能返回：`authorization_pending`、`slow_down`、`access_denied`、`invalid_grant`。

## 9. 资源服务校验 access token

### 9.1 本地验签（高吞吐）

1. 拉取 discovery：`/.well-known/openid-configuration`
2. 拉取 JWKS：`/oauth2/jwks`
3. 校验 `iss`、`aud`、`exp`、`nbf`、`scp`

### 9.2 回源 introspection（强一致）

```bash
curl -X POST "http://www.idpsienne.uk/oauth2/introspect" \
  -H "Authorization: Basic $(printf 'service-client:service123' | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "token=ACCESS_TOKEN"
```

注意：`token_endpoint_auth_method=none` 的客户端不能用于 introspection。

## 10. UserInfo 获取用户资料

```bash
curl "http://www.idpsienne.uk/oauth2/userinfo" \
  -H "Authorization: Bearer ACCESS_TOKEN"
```

## 11. 登出集成

本地会话登出：

- `POST /logout`（当前会话）
- `POST /logout/all`（当前用户所有会话）

OIDC RP-Initiated Logout：

- `GET /connect/logout?client_id=...&post_logout_redirect_uri=...&state=...` 获取确认页和 CSRF
- `POST /connect/logout` 执行注销并回跳

## 12. 动态客户端管理（内部接口）

- `POST /oauth2/clients`
- `POST /oauth2/clients/{client_id}/redirect-uris`
- `POST /oauth2/clients/{client_id}/post-logout-redirect-uris`

当前路由未挂鉴权中间件，请仅在内网管理面使用，禁止直接暴露公网。

## 13. 常见失败与定位

- `invalid_request`：参数缺失或格式错误
- `invalid_client`：client 认证失败、状态非 active、auth method 不匹配
- `invalid_grant`：授权码/refresh token/device_code 无效或过期
- `invalid_scope`：请求 scope 超出 client 允许范围
- `unsupported_grant_type`：grant 类型未启用或拼写错误
- `invalid return_to`：`return_to` 不是本地路径
- `invalid csrf token`：浏览器交互 POST 缺少双提交 token
